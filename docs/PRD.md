# 个性化全科学习平台 · 产品需求文档

> 版本：v1.0 · 2026-06-28
> 目标用户：1 名武汉初二学生（2026年9月升初三），通过中考（2027年6月）
> 定位：完全替代课外辅导班的私有个性化学习平台

---

## 一、项目概述

### 1.1 背景

为一名武汉市初中生打造的全科自学平台，覆盖中考9科（语数英物化生地史道法），从初二暑假持续使用至中考结束。系统基于官方课标知识图谱，结合武汉中考考频权重，提供个性化学习路径、AI辅导、智能练习和家长监控。

### 1.2 核心原则

- 全科覆盖，不做减法
- 数据驱动：所有内容基于课标知识树，按武汉中考考频排优先级
- 复用 TeachAny 官方课件，自托管为静态文件
- 运行时不依赖外部 Agent，只用自建 API + MiniMax AI
- 面向单用户，所有视图带个人学情色彩

### 1.3 技术栈

| 层 | 技术 |
|---|---|
| 后端 | Python FastAPI + SQLite (aiosqlite) |
| 前端 | Vue 3 + TypeScript + Vite |
| AI | MiniMax M3 (OpenAI 兼容 API) |
| TTS | Edge TTS（微软神经语音）|
| 部署 | Docker Compose + Nginx + cloudflared 隧道 (端口8088) |

## 三、功能模块

### 3.1 全科知识图谱

**路由**：`/graph`（全科总览）、`/map/:subject`（单科地图）

- 9科官方课标知识树，共 344 个知识节点
- 节点间有前置依赖关系（prerequisites）
- 每个节点带掌握度（红/黄/绿三色）
- 武汉中考考频权重标注（高/中/低）
- 全科图谱一屏展示所有学科进度

### 3.2 互动课件

**路由**：`/learn/:nodeId`、`/library`（课件库）

- 311 个自托管互动课件（8科，从 TeachAny 拉取）
- 每个课件含：互动教学页 + TTS 语音讲解 + AI 学伴
- AI 学伴：MiniMax M3，流式对话，运行时可热切换模型/密钥
- 课件库按学科/年级/考频浏览和筛选

### 3.3 学习路径

**路由**：`/path`

- 基于知识依赖图 + 掌握度 + 考频自动生成个性化路径
- 状态机：已掌握 → 学习中 → 待解锁
- 推荐下一步学习内容

### 3.4 智能练习

**路由**：`/practice`、`/practice/:nodeId`

- 题库来源：
  - AI 生成基础题 2783 道（9科）
  - 真题提取：441 份 PDF 试卷多模态处理（MiniMax M3 视觉识别）
- 支持题型：选择、填空、简答、证明、实验
- 自适应选题：按掌握度+考频+难度
- 公式渲染：KaTeX（`$...$` LaTeX）
- 图片支持：题目配图（试卷原页截图 WebP）+ 图形文字描述
- 答题后即时反馈 + 错因分析

### 3.5 错题本

**路由**：`/errors`

- 答错的题自动进入错题本
- 按艾宾浩斯间隔复习
- AI 生成详细讲解（首次生成 + 缓存 HTML）
- 练习页和错题本双入口查看讲解

### 3.6 学情诊断

**路由**：`/diagnosis`

- 按知识点粒度定位薄弱环节
- 自动回溯前置知识缺口
- 诊断结果影响学习路径推荐

### 3.7 艾宾浩斯记忆系统

**路由**：`/memory`

- 6级间隔复习：1→3→7→15→30天→永久掌握
- 两种训练模式：翻卡模式 + 拼写模式
- 学科覆盖：英语词汇/语法、历史事件、政治要点、语文古诗词
- 答错降回第1级，连续3次正确跳级
- 家长可设置每日新卡数量（5-100，默认20）
- 卡片数据：AI 批量生成约 3300 张 + Edge TTS 英语发音音频

### 3.8 家长任务系统

**路由**：学生首页显示 + 家长端 `/parent` 布置

- 任务类型：做题 / 看课件 / 背诵 / 自定义
- 任务属性：标题、学科、知识点、目标数量、截止日期、循环频率、备注
- 循环任务：每天/每周自动生成下一期
- 过期任务：家长端+学生端均标红
- 任务→记忆关联：背诵类任务跳转到 `/memory`

### 3.9 家长看板

**路由**：`/parent`

- 每日学习报告（做题数、正确率、学习时长）
- 各科掌握度进度可视化
- AI 配置面板（运行时切换模型/密钥/端点）
- 学科开关（隐藏的学科在学生端全部不可见）
- 课件制作入口（调 TeachAny Skill 生成新课件）
- 任务布置和执行监控

### 3.10 PBL 探究

**路由**：`/pbl`

- 项目式学习课题（跨学科探究）
- 自托管 PBL 课件
- AI 分析辅助

### 3.11 学习计划

**路由**：首页 `/`

- 自动生成每日学习计划
- 结合诊断结果 + 任务系统 + 复习安排

### 3.12 进度可视化

**路由**：`/progress`

- 各科掌握度柱状图
- 学习时间统计
- 答题正确率趋势

## 四、数据来源

### 4.1 知识图谱

- 来源：TeachAny 官方课标树（9科 344 节点）
- 武汉中考考频校准（`tools/wuhan_weights.py`）
- 人教版教材对齐

### 4.2 课件

- 来源：TeachAny 社区课件，自动拉取并自托管
- 数量：311 个互动课件
- 格式：HTML + CSS + JS + TTS(mp3) + 图片

### 4.3 题库

| 来源 | 数量 | 特点 |
|------|------|------|
| AI 生成 | 2783 道 | 覆盖9科，基础题为主 |
| 真题提取 | ~5000 道(预期) | 441 份 PDF 多模态提取，含公式/图片/图形描述 |

### 4.4 记忆卡片

- AI 批量生成约 3300 张
- 英语词汇/语法、历史事件、政治要点、古诗词
- 英语单词配 Edge TTS 发音音频

### 4.5 试卷

- 441 份武汉中考/模拟/调考 PDF
- 多模态处理：逐页渲染 → MiniMax M3 视觉识别 → JSON + 页面截图(WebP)

## 五、部署架构

```
用户设备 (浏览器)
    ↓ HTTPS
cloudflared 隧道 (端口 8088)
    ↓
Nginx (反向代理)
    ├─ /api/*        → FastAPI 后端 (端口 8000)
    ├─ /courseware/* → 静态课件文件
    ├─ /exam-pages/* → 试卷页面截图
    └─ /*           → Vue 前端 (dist/)
```

## 九、非功能需求

| 项 | 要求 |
|---|---|
| 性能 | 页面加载 < 2s，API 响应 < 500ms（除AI调用外）|
| 数据安全 | SQLite 本地存储，不上传任何学习数据到第三方 |
| 可用性 | 7×24 运行，cloudflared 自动重连 |
| 兼容性 | Chrome/Edge/Safari，支持手机/平板/电脑 |
| 隐私 | 不显示任何 "TeachAny" 品牌标识 |
